Windows Azure remote access page

This dialog allows you to enable or disable remote access for all the roles in your project at the same time.

To enable remote access, select the Enable checkbox and specify the login credentials that will be used for connecting a Remote Desktop client to a role instance within your Windows Azure cloud.

The Certificate is needed to encrypt your remote access password. You may specify an existing X.509 certificate file or create a new self-signed certificate from this dialog. If specifying an existing one, you need both the public-key containing certificate file (typically with a *.CER extension) and a private-key containing version of the same certificate encoded using the SHA1, PKCS#12 format (typically with a .pfx extension). The public key from the *.CER certificate file will be used to encrypt your remote access password inside your configuration file (*.CSCFG). The private key-containing PFX file is what you need to upload to your hosted service using the Windows Azure portal.

To disable remote access, deselect the Enable checkbox.

To learn more, see the Microsoft Interoperability Bridges website: http://java.interopbridges.com/cloud.
